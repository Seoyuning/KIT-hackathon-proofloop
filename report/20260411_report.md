### 2-5. `Invalid API key` 대응

사용자 리포트: 로그인 시 `Invalid API key`.

원인 진단: `curl`로 Supabase `/auth/v1/settings`에 anon 키를 직접 쳐서 200 정상 응답 확인 → **키 자체는 유효**. `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`가 Vercel에 없거나 인라인되지 않은 배포가 서비스 중인 상태로 추정.

안내 내용:

1. Vercel → Settings → Environment Variables에 두 키 추가(Production/Preview/Development 모두)
2. `NEXT_PUBLIC_*`는 빌드 시 JS 번들에 인라인되므로 **재배포 필수**
3. 값 붙여넣을 때 따옴표 포함 금지
4. Supabase → Authentication → URL Configuration에 Site URL과 `/auth/callback` 등록

### 2-6. `/studio` 멈춤 해소

**증상**: 교사로 가입 후 `/studio`에서 스플래시만 뜨고 영원히 멈춤.

**원인 분석**: 구버전 `src/app/studio/page.tsx`가 1.35초짜리 페이드 애니메이션을 돌리고, 그 다음에야 `!done || isLoading` 가드 뒤에서 역할 기반 `router.replace`를 실행. `supabase.auth.getSession()`이 어떤 이유로든 throw하면 `isLoading`이 영원히 `true`로 남아 리디렉트가 발동하지 않음. 에러 핸들링도 없어서 디버깅 단서조차 없었다.

**조치**:

- `src/app/studio/page.tsx`: 애니메이션 상태(`visible`/`done`)와 피처 카드 제거. "이동 중…" 한 줄만 보여주고 `isLoading === false`가 되는 즉시 `router.replace`.
- `src/lib/auth-context.tsx`: `getSession()`과 `onAuthStateChange` 핸들러를 `try/catch/finally`로 감싸 **항상 `isLoading`을 `false`로 떨어뜨리고**, 에러는 `console.error("[auth] ...")`로 남김.

이후 사용자가 옛 스플래시 화면 스크린샷을 보냈을 때, 이는 Vercel 재배포 전 캐시된 화면임을 확인해 하드 리프레시와 재배포 대기를 안내.

참고로 한 번 더 블로커가 있었는데, `/studio/login`의 `useSearchParams()`가 `Suspense` 경계 없이 쓰여 **production 빌드 전체가 export 단계에서 실패**했다. 이것이 "사이트가 아예 안 뜬다"의 진짜 원인이었다. 내부 폼을 `LoginForm`으로 분리하고 `<Suspense fallback={null}>`로 감싸 해결.

### 2-7. 한글 라벨 줄바꿈 버그

**증상**: "로그아웃" 버튼이 "로그아" + 줄바꿈 + "웃"으로 깨짐.

**원인**: `src/app/globals.css`의 `overflow-wrap: anywhere`. `word-break: keep-all`과 조합되었지만, `anywhere`는 overflow 압력이 생기면 Hangul 음절 사이 어디에서든 soft break를 허용한다.

**조치**:

1. `overflow-wrap: anywhere` → `break-word`로 전역 변경. `break-word`는 진짜 overflow가 날 때만, 그리고 단어 경계를 우선으로 끊는다.
2. 절대 줄바꿈되면 안 되는 짧은 한글 라벨에 `whitespace-nowrap`을 명시적으로 추가:
   - `studio/layout.tsx`: 로그아웃 버튼 + 사이드바 네비 링크(질문하기 / 마이페이지 / 질문 분석 / 수업 도구)
   - `studio/login/page.tsx`: 제출 버튼
   - `studio/mypage/page.tsx`: 이름 저장 / 비밀번호 변경 / 로그아웃
   - `studio/analysis/page.tsx`: "질문 분석" pill, 교과서 pill
   - `studio/generate/page.tsx`: "수업 도구" pill, 교과서 pill, `TeacherModeButton`, 생성 버튼
   - `studio/chat/page.tsx`: "질문 보내기" 버튼, starter-prompt pill

긴 문장(설명 카피, FAQ 답변 등)은 `word-break: keep-all + break-word` 조합으로 자연스럽게 단어 단위 줄바꿈이 되도록 두고 `nowrap`을 넣지 않았다.

## 2. 세부 작업 내용

### 2-1. 개발 서버 실행 및 환경 정리

- `package.json`과 `package-lock.json`을 확인해 npm 기반 프로젝트임을 파악했다.
- `node_modules`가 없어 의존성 설치를 진행했다.
- PowerShell 실행 정책 때문에 `npm` 대신 `npm.cmd`를 사용했다.
- 샌드박스 권한 문제로 `npm install` 및 `next dev` 실행에 승인이 필요했고, 승인 후 설치와 서버 실행을 완료했다.
- `http://localhost:3000`에서 실제 `200` 응답을 확인했다.

### 2-2. 기존 서비스 구조 분석

- 기존 루트 페이지는 소개/설명과 시연 화면이 한 페이지에 합쳐진 구조였다.
- 별도 사용자용 서비스 URL이 없고 `/`에서 모든 기능이 노출되는 상태였다.
- `/api/diagnose`를 사용하는 초기 학습 부채 진단 MVP가 존재했다.

### 2-3. 실제 서비스 라우트 분리

- 실제 사용 화면을 `/studio`로 분리했다.
- `/`는 제품 설명과 포지셔닝 중심의 랜딩으로 재구성했다.
- 이 과정에서 “코칭 순서”라는 표현을 “점검 우선순위”로 바꾸고, 제품의 목적을 더 현실적인 운영 도구로 재정의했다.

### 2-4. 제품 메시지 조정

- “학원 교강사 전체”가 아니라 학생 상태를 운영하는 역할에 맞는 도구라는 방향으로 타겟을 좁혔다.
- 담임형 강사, 튜터, TA, 학습 매니저처럼 학생 질문과 과제 검토 이후 후속 조치를 수행하는 역할에 더 적합하다는 메시지를 반영했다.
- 순수 강의형 역할에는 덜 맞는다는 점도 명시했다.

### 2-5. 이미지 기반 새 서비스 구현

사용자 제공 이미지의 구조를 반영해 `/studio`를 새 서비스로 구현했다.

#### 학생 영역

- 과목별 / 출판사별 교과서 봇 선택
- 교과서 단원과 쪽수 근거를 포함한 답변 생성
- 추천 시작 질문 제공
- 학생 질문 입력 및 대화형 응답

#### 통합 질문 DB

- 학생 질문을 단원별 질문 클러스터로 누적
- 오개념, 학생 니즈, 교사용 액션을 함께 관리
- 질문 빈도를 기반으로 상위 질문을 집계

#### 교사 영역

- 강의자료 초안 생성
- 시험지 초안 생성
- 질문 DB 기반 오개념 반영
- 수업 범위 / 시험 범위 선택 및 생성 결과 표시

### 2-6. 데이터 및 생성 로직 분리

새 서비스 구조를 위해 로컬 데이터와 생성 로직을 별도 파일로 분리했다.

- `src/lib/studio-data.ts`
  - 교과서 봇 데이터
  - 단원 데이터
  - 초기 질문 클러스터 데이터
- `src/lib/studio-generation.ts`
  - 교과서 기반 답변 생성
  - 질문 DB 누적 로직
  - 강의자료 생성 로직
  - 시험지 초안 생성 로직

### 2-7. `/studio` UI를 실제 사용형 앱으로 재설계

초기 `/studio`는 여전히 소개형 웹사이트 느낌이 강했다.

이에 따라 다음과 같이 재설계했다.

- 좌측 사이드바
  - 교과서 봇 목록
  - 현재 상태 요약
  - 시작 질문 / 상위 질문
- 중앙 메인 영역
  - 학생 챗봇 대화
  - 통합 질문 DB
- 우측 작업 패널
  - 교과서 범위 카드
  - 교사용 생성기
  - 강의자료 / 시험지 초안 모드 전환

설명형 히어로 섹션을 줄이고, 바로 작업 가능한 앱 구조로 재배치했다.

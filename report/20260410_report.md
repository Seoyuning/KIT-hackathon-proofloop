## 2. 세부 작업 내용

### 2-1. 메인 페이지 UI 분석

기존 `/studio`는 사이드바 + 메인 영역에 다음이 모두 한 화면에 표시되어 있었다:
- 학생용 챗봇
- 질문 DB (통합 질문 클러스터)
- 교과서 범위 카드
- 교사용 생성기 (강의 자료 / 시험지 초안)

정보 과부하로 가독성이 떨어지는 문제가 있었다.

### 2-2. 역할별 페이지 분리

한 화면 통합 구조를 역할별로 완전히 분리했다:

| 라우트 | 역할 | 내용 |
|--------|------|------|
| `/studio` | 공통 | 인증 상태에 따라 자동 리다이렉트 |
| `/studio/login` | 공통 | 로그인 / 회원가입 |
| `/studio/chat` | 학생 전용 | 교과서 챗봇 대화 |
| `/studio/analysis` | 교사 전용 | 질문 DB + 교과서 범위 분석 |
| `/studio/generate` | 교사 전용 | 강의 자료 / 시험지 초안 생성 |

#### 구현 방식

- **React Context (`StudioProvider`)**: 봇 선택, 채팅, 질문 DB, 교사 생성 상태를 공유
- **React Context (`AuthProvider`)**: 인증 상태 (이메일, 비밀번호, 역할) 관리
- **공유 레이아웃 (`layout.tsx`)**: 역할별 사이드바 네비게이션, 로그인 페이지에서는 사이드바 숨김
- **접근 제어**: 각 페이지에서 `useAuth()` 확인 후 미인증/역할 불일치 시 리다이렉트

### 2-3. 인증 시스템 구현

처음에는 이름만으로 로그인하는 구조였으나, 이메일+비밀번호 방식으로 변경했다.

#### 회원가입

- 이메일 (형식 검증)
- 비밀번호 (4자 이상)
- 이름
- 역할 선택 (학생 / 교사) — 가입 시 결정, 이후 변경 불가

#### 로그인

- 이메일 + 비밀번호
- 가입 시 선택한 역할에 따라 자동 라우팅

#### 기술 세부사항

- localStorage 기반 MVP (서버 사이드 인증 아님)
- 비밀번호는 간단한 해시로 저장 (프로덕션용이 아닌 MVP)
- 나중에 SNS OAuth (카카오, 구글 등) 연동 가능한 구조

### 2-4. 접근 제어 흐름

| 상태 | `/studio` 접근 시 |
|------|-------------------|
| 미로그인 | → `/studio/login` |
| 학생 로그인 | → `/studio/chat` |
| 교사 로그인 | → `/studio/analysis` |
| 학생이 교사 페이지 접근 | → `/studio/chat`으로 리다이렉트 |
| 교사가 학생 페이지 접근 | → `/studio/analysis`로 리다이렉트 |

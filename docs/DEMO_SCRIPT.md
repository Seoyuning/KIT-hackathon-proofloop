# ProofLoop 90초 데모 스크립트

## 데모 목표

심사위원에게 이 메시지를 빠르게 전달한다.

> "학생은 AI를 자유롭게 쓸 수 있지만, AI가 매 대화마다 학생을 검증한다. 과제를 AI에게 시켜도 AI가 오히려 학생을 검증하는 역전이 일어난다. 그리고 그 데이터가 그대로 교사에게 흘러가 수업이 된다."

## 권장 데모 흐름

### 1. 오프닝, 15초

"AI 시대에 학생들은 AI를 학습에 쓰지만, 과제 대행 수단으로도 씁니다. 교사는 학생이 뭘 모르는지 데이터로 파악할 경로가 없습니다. ProofLoop는 학생이 AI를 쓸수록 오히려 학습이 일어나고, 그 데이터가 교사의 수업이 되는 교과서 기반 학습 루프입니다."

### 2. 회원가입 / 역할+학년 선택, 10초

`/studio/login`을 보여주며 말한다.

"가입할 때 학생/교사 역할과 학년을 선택합니다. 학생은 중1부터 고3까지 자기 학년을 고르고, 이후 역할에 맞는 화면만 보게 됩니다."

### 3. 교사: 반 만들기 + 초대 코드, 15초

교사 계정으로 `/studio/classes`에서 반을 생성한다.

"교사가 반을 만들면 6자리 초대 코드가 자동 생성됩니다. 코드를 클릭하면 바로 복사됩니다. 이 코드를 학생에게 카카오톡으로 보내면, 학생은 입력만 하면 반에 참여할 수 있습니다. 반 생성 시 학년, 과목, 출판사, 교과서를 지정합니다."

### 4. 학생: 반 참여 + 챗봇 질문, 25초

학생 계정으로 `/studio/chat`에서 초대 코드로 반에 참여하고 질문을 던진다.

"학생이 반에 참여하면, 해당 반의 교과서에 맞는 챗봇이 자동으로 활성화됩니다."

질문을 하나 보낸 후 결과를 보여주며 말한다.

"답변을 보세요. 세 가지가 동시에 나옵니다. 교과서 단원/쪽수 근거, 이해도 1~5점 측정, 그리고 서술형 후속 질문입니다. 이건 일반 질문에도 매번 나옵니다. 학생이 AI를 아무리 써도 진짜 이해 없이는 빠져나갈 수 없습니다."

이어서 "이 문제 풀어줘"를 입력한다.

"과제를 AI에게 시키면 어떻게 되는지 보겠습니다. AI가 답을 거절하고, 대신 핵심 개념을 설명한 뒤 서술형 질문을 냅니다. 과제를 맡겨도 AI가 학생을 역으로 검증하는 겁니다."

강조할 포인트:
- 매 답변: 근거 + 이해도 + 후속 질문 (일반 질문에도 항상)
- 과제 대행 시도 → AI가 거절하고 학생을 검증
- 이 모든 데이터가 DB에 자동 저장

### 5. 교사: 대시보드 + 학생별 약점, 15초

교사 계정으로 `/studio/analysis`를 보여준다.

"교사 화면으로 넘어오면, 방금 학생이 한 질문이 이미 데이터로 집계되어 있습니다. 누적 질문 수, 참여 학생 수, 상위 오개념, 평균 이해도를 한눈에 봅니다. 반 관리 페이지에서는 학생별로 약점 개념과 오개념 태그까지 확인할 수 있습니다."

### 6. 교사: 수업 자료 / 시험 생성 + 내보내기, 15초

`/studio/generate`로 이동한다.

"이 데이터를 기반으로 수업 자료와 시험 초안을 바로 만들 수 있습니다. 학생들이 실제로 어려워한 지점이 반영된 자료입니다. 결과물은 클립보드 복사나 파일 다운로드로 즉시 내보낼 수 있습니다."

### 7. 마무리, 5초

"학생이 AI를 쓸수록 학습이 일어나고, 그 데이터가 교사의 수업이 된다 — ProofLoop."

## 심사위원 질문 예상 답변

### Q. 학생이 AI로 과제를 대행하는 문제는 어떻게 해결하나요?

"AI 사용을 막는 게 아니라, AI가 매 대화에서 학생을 역으로 검증합니다. 일반 질문에도 매번 이해도를 측정하고 서술형 후속 질문을 냅니다. 과제를 시키면 답 대신 개념 설명과 확인 질문을 줍니다. AI를 쓸수록 학습이 일어나는 구조입니다."

### Q. 왜 기존 LMS나 AI 튜터로 안 되나요?

"LMS는 출결과 과제 관리에 강하지만 학생이 어디서 막혔는지 실데이터로 보여주지 못합니다. 범용 AI 튜터는 교과서 범위를 벗어나기 쉽고 교사 측과 연결되지 않습니다. ProofLoop는 교과서 범위 고정 + 지속적 검증 + 학생-교사 데이터 연결, 세 가지를 동시에 해결합니다."

### Q. 기술적으로 어떻게 구현했나요?

"Gemini 2.5 Flash에 교과서 단원 데이터를 system instruction으로 전량 주입합니다. 매 답변에서 [근거], [이해도], [후속 질문] 태그를 강제 출력하도록 구조화했고, 파싱 후 Supabase DB에 저장됩니다. 교사 대시보드는 이 데이터를 실시간 집계합니다. 인증은 Supabase 서버 프록시 방식으로 처리합니다."

### Q. 교과서 데이터는 어떻게 확보했나요?

"55종 실제 한국 교과서(10개 출판사, 15개 과목)의 단원 구조를 정적 카탈로그로 탑재했습니다. 해커톤 기간 내 PDF OCR 파이프라인 완성이 어렵다고 판단해, 안정성을 우선으로 정적 데이터를 선택했습니다."

### Q. 지금 어떤 한계가 있나요?

"교과서 내용 자체의 전문 탑재는 다음 단계이고, 현재는 단원 구조와 핵심 개념 수준입니다. 향후 교과서 PDF 업로드와 벡터 DB 기반 검색을 추가하면 답변 품질이 크게 올라갈 수 있습니다."

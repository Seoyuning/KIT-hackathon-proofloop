# ProofLoop 90-Second Demo Script

## 데모 목표

심사위원에게 이 메시지를 빠르게 전달한다.

> "학생은 교과서 범위 안에서 질문하고, 교사는 그 질문 데이터로 수업을 준비한다. ProofLoop는 학생과 교사를 같은 교과서 위에서 연결한다."

## 권장 데모 흐름

### 1. 오프닝, 15초

"AI 시대에도 공교육과 학원 수업의 중심은 여전히 교과서입니다. 그런데 학생이 묻는 질문과 교사가 준비하는 수업은 서로 완전히 단절되어 있습니다. ProofLoop는 그 두 흐름을 같은 교과서와 같은 질문 DB 위에서 연결하는 학습 스튜디오입니다."

### 2. 로그인 / 역할 선택, 10초

`/studio/login`을 보여주며 말한다.

"가입할 때 학생과 교사 중 역할을 선택합니다. 이후 각 사용자는 자기 역할에 필요한 화면만 보게 됩니다. 먼저 학생으로 들어가 보겠습니다."

### 3. 학생 챗봇, 25초

`/studio/chat`으로 이동해 교과서 봇을 선택하고 질문을 하나 던진다.

"학년, 과목, 출판사 기준으로 교과서 봇을 고를 수 있습니다. 이 챗봇은 선택한 교과서 범위를 벗어나지 않습니다. 답변에는 단원과 페이지 근거가 함께 나옵니다. 그리고 이 질문은 자동으로 반 전체의 통합 질문 DB에 쌓입니다."

강조할 포인트:

- 교과서 범위에 고정된 답
- 단원/페이지 근거 제시
- 질문이 자동으로 교사 측 데이터로 흘러감

### 4. 교사 분석, 20초

교사 계정으로 로그인하거나 전환해 `/studio/analysis`를 보여준다.

"교사 화면으로 넘어오면, 학생들의 질문이 단원별 클러스터로 정리되어 있습니다. 어떤 단원에 질문이 몰리는지, 어떤 개념에서 막히는지 한눈에 보입니다. 교사는 감이 아니라 데이터로 수업을 준비합니다."

강조할 포인트:

- 반 전체가 어디서 막혔는지가 데이터로 보인다
- 학생 질문이 그대로 교사의 입력이 된다

### 5. 수업 자료 / 시험 초안 생성, 20초

`/studio/generate`로 이동한다.

"선택한 교과서 범위를 기준으로, 수업 자료 초안과 시험 초안을 바로 만들 수 있습니다. 학생들이 실제로 어려워한 지점을 반영한 자료라서, 감으로 만든 자료와 완전히 다릅니다."

### 6. 마무리, 10초

"ProofLoop는 LMS가 아닙니다. 범용 AI 튜터도 아닙니다. 학생과 교사를 같은 교과서 데이터 위에서 연결하는, 교과서 기반 학습 스튜디오입니다."

## 10초 버전 한 줄 마무리

"학생이 교과서 안에서 묻고, 교사가 그 데이터로 수업을 만든다 — ProofLoop."

## 심사위원 질문 예상 답변

### Q. 왜 기존 LMS나 AI 튜터로 안 되나요?

"LMS는 출결과 과제 관리에 강하지만 학생이 어디서 막혔는지 실데이터로 보여주지 못합니다. 범용 AI 튜터는 교과서 범위를 벗어나기 쉽고 교사 측과 연결되지 않습니다. ProofLoop는 교과서 범위 고정과 학생-교사 데이터 연결, 두 가지를 동시에 해결합니다."

### Q. 왜 이게 지금 중요한가요?

"공교육과 학원 모두 교재 기반 수업이 여전히 중심이지만, 학생의 질문 데이터를 교사의 수업 준비에 연결하는 제품은 드뭅니다. AI 확산 이후 이 공백이 더 뚜렷해졌습니다."

### Q. 실제 현장에서 누가 쓰나요?

"학생은 교과서 범위 안에서 질문하고, 교사는 그 질문 데이터로 수업 자료와 시험을 준비합니다. 학원 강사와 공교육 교사 모두 같은 흐름으로 사용할 수 있습니다."

### Q. 지금 어떤 한계가 있나요?

"현재는 교과서 시드 데이터(수학/과학/국어) 기반 MVP이고, 인증은 localStorage, 질문 DB는 인메모리 구조입니다. 다음 단계로 교과서 PDF 업로드, 실제 DB 연동, 서버 사이드 인증을 계획하고 있습니다."

## 데모 직전 체크

- 로컬은 `npm run dev` 후 `http://localhost:3000/studio/login` 열기
- 학생 / 교사 테스트 계정이 각각 준비돼 있는지 확인
- 학생 화면에서 교과서 봇이 정상 선택되는지 확인
- 교사 화면에서 질문 클러스터와 생성 기능이 동작하는지 확인
- 데모 중 역할 전환 흐름(학생 → 교사)을 미리 한 번 리허설할 것

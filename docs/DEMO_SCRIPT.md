## 체험 순서 (교사 → 학생 → 교사 확인)

### STEP 1. 교사 로그인 + 반 만들기

1. 배포 URL 접속 → `/studio/login`으로 이동
2. **교사 계정**으로 로그인
3. 좌측 사이드바에 "반 만들러 가기 →" 클릭 또는 상단 "반 관리" 탭 클릭
4. **"+ 반 만들기"** 버튼 클릭
5. 아래와 같이 입력:
   - 반 이름: `1학년 3반`
   - 학년: `고1`
   - 과목: `수학`
   - 출판사: `비상교육`
   - 교과서명: `수학 I`
6. **"반 만들기"** 클릭
7. 생성된 반의 **6자리 초대 코드**를 클릭하면 자동 복사됨 (메모해 두세요)

### STEP 2. 학생 로그인 + 반 참여 + 질문하기

1. **로그아웃** (사이드바 상단 "로그아웃" 버튼)
2. **학생 계정**으로 로그인
3. 좌측 사이드바 "반 참여" 영역에 **STEP 1에서 복사한 초대 코드** 입력 → "참여" 클릭
4. 반에 참여하면 챗봇이 활성화됩니다. 아래 질문을 보내보세요:

**질문 1 (일반 질문):**
```
이차함수의 꼭짓점이 뭐야?
```
→ 답변에 **교과서 근거 카드** + **이해도 바** + **서술형 후속 질문**이 함께 표시됩니다.

**질문 2 (과제 대행 시도):**
```
이 문제 풀어줘: y = -2(x-1)^2 + 3의 꼭짓점을 구하시오
```
→ AI가 **답을 거절**하고, 핵심 개념을 설명한 뒤 **서술형 확인 질문**을 냅니다.

**질문 3 (추가 질문):**
```
순열과 조합의 차이를 교과서 기준으로 설명해줘
```

### STEP 3. 교사로 다시 로그인 + 대시보드 확인

1. **로그아웃** 후 **교사 계정**으로 다시 로그인
2. 사이드바 "내 반"에서 **"1학년 3반"** 클릭
3. **"질문 분석"** 탭에서 확인:
   - 상단 카드: 누적 질문 수, 참여 학생 수가 증가했음
   - 단원별 평균 이해도: 학생의 이해도 점수가 집계됨
4. **"반 관리"** 탭에서 확인:
   - "1학년 3반" 클릭 → 학생 이름, 질문 횟수, 약점 개념이 표시됨
5. **"수업 도구"** 탭에서 확인:
   - 수업 자료 생성 / 시험 초안 생성 가능
   - 클립보드 복사 또는 파일 다운로드로 내보내기

## 핵심 포인트 요약

| 체험 포인트 | 어디서 확인 |
|---|---|
| AI가 매 답변마다 이해도를 측정한다 | 학생 챗봇 → 답변 하단 이해도 바 |
| AI가 매 답변마다 서술형 후속 질문을 낸다 | 학생 챗봇 → 답변 하단 틸색 카드 |
| "풀어줘" 하면 AI가 거절한다 | 학생 챗봇 → STEP 2 질문 2 |
| 학생 데이터가 교사 대시보드에 실시간 반영된다 | 교사 질문 분석 → 누적 질문, 이해도 |
| 학생별 약점을 개별 추적한다 | 교사 반 관리 → 학생 클릭 |
| 수업 자료/시험을 데이터 기반으로 생성한다 | 교사 수업 도구 |

## 90초 발표 스크립트

### 오프닝 (15초)
"AI 시대에 학생들은 AI로 과제를 대행합니다. 교사는 학생이 뭘 모르는지 감으로 파악합니다. ProofLoop는 AI 사용 자체가 학습 행위가 되고, 그 데이터가 교사의 수업이 되는 구조입니다."

### 교사: 반 만들기 (10초)
"교사가 반을 만들면 6자리 초대 코드가 생성됩니다. 코드를 학생에게 공유합니다."

### 학생: 챗봇 질문 (25초)
"학생이 질문하면 세 가지가 동시에 나옵니다. 교과서 근거, 이해도 측정, 서술형 후속 질문. '풀어줘'라고 하면 AI가 답을 거절하고 학생을 검증합니다."

### 교사: 대시보드 (20초)
"교사 화면으로 돌아오면 방금 학생이 한 질문이 이미 데이터로 집계되어 있습니다. 학생별 약점, 단원별 이해도를 실시간으로 확인합니다."

### 교사: 수업 자료 생성 (15초)
"이 데이터로 수업 자료와 시험 초안을 바로 만듭니다. 클립보드 복사나 파일 다운로드로 즉시 내보냅니다."

### 마무리 (5초)
"학생이 AI를 쓸수록 학습이 일어나고, 그 데이터가 교사의 수업이 된다 — ProofLoop."

## 심사위원 예상 질문

**Q. 학생이 AI로 과제를 대행하는 문제는?**
"AI 사용을 막는 게 아니라, AI가 매 대화에서 학생을 역으로 검증합니다. 일반 질문에도 매번 이해도를 측정하고 서술형 질문을 냅니다. AI를 쓸수록 학습이 일어나는 구조입니다."

**Q. 교과서 데이터는 어떻게 확보했나요?**
"현재는 주요 교과서의 단원 구조를 직접 탑재했습니다. 향후 출판사 API 연동(비상교육 비바샘, 미래엔 엠티처 등) 또는 교육부 디지털교과서 플랫폼 연계로 자동화를 계획하고 있습니다."

**Q. 기술적으로 어떻게 구현했나요?**
"Gemini 2.5 Flash에 교과서 단원 데이터를 system instruction으로 주입합니다. 매 답변에서 [근거], [이해도], [후속 질문] 태그를 강제 출력하고, 파싱 후 Supabase DB에 저장합니다. 교사 대시보드는 이 데이터를 실시간 집계합니다."

# AI 빌딩 리포트 초안

아래 내용은 제출 양식에 바로 옮겨 적을 수 있도록 정리한 초안입니다.  
팀명, 이름, 휴대폰번호만 추가하면 됩니다.

## 프로젝트명

ProofLoop

## 1. 기획

### 설정한 사용자는 누구이며, 해결하려는 구체적인 문제점/불편함은 무엇인가요?

주 사용자는 코딩 교육 기관의 교강사입니다.  
보조 사용자는 수강생과 교육 운영자입니다.

생성형 AI가 보편화되면서 수강생은 과제를 더 빠르게 완성할 수 있게 되었지만, 교강사는 그 결과물이 학습자의 실제 이해를 반영하는지 판단하기 더 어려워졌습니다. 표면적으로는 과제가 완료된 것처럼 보여도, 다음 과제나 실습 단계에서 개념이 무너지는 경우가 많고, 이때는 이미 학습 부채가 누적된 뒤입니다.

ProofLoop는 이 문제를 해결하기 위해 과제 설명, 학생 제출물, AI 대화 흔적을 함께 읽고 "이 학생이 실제로 이해했는가"를 진단하는 AI 기반 학습 증거 레이더를 제공합니다.

### 문제를 해결하기 위한 솔루션의 핵심 기능은 무엇인가요?

1. 학습 증거 진단
   과제 설명, 제출물, AI 대화 흔적을 바탕으로 이해 증거 점수를 생성합니다.

2. 위험 신호 탐지
   결과물은 완성됐지만 개념 설명이 빈약하거나 AI 의존도가 높은 경우를 탐지합니다.

3. 적응형 방어 질문 생성
   교강사가 바로 사용할 수 있는 구두 점검 질문과 수정 미션을 생성합니다.

4. 코호트 우선순위 대시보드
   한 반 전체를 학습 증거 기준으로 정렬해 누가 즉시 개입 대상인지 보여줍니다.

### 이 솔루션이 도입되었을 때 기대되는 개선점이 무엇인가요?

- 교강사는 과제 결과물만 보고 판단하던 한계를 줄이고, 실제 이해 여부를 빠르게 검증할 수 있습니다.
- 수강생은 "과제를 끝내는 것"이 아니라 "이해를 증명하는 것"을 목표로 학습하게 됩니다.
- 교육 운영자는 학습 부채와 이탈 위험을 더 이른 시점에 발견할 수 있습니다.
- AI를 학습 보조 도구로 활용하면서도, 교육 품질과 피드백 정확도를 함께 높일 수 있습니다.

## AI 활용 전략

### 이 프로젝트에서 사용할 AI 도구와 모델은 무엇이며, 선택한 이유는 무엇인가요?

- Codex 기반 코딩 에이전트
  프로젝트 초기 아이디어 정제, MVP 구조 설계, 프론트엔드 구현, 문서화까지 빠르게 일관된 품질로 진행하기 위해 사용했습니다.

- Gemini API 기반 런타임 분석 레이어
  실서비스 모드에서 학습자 제출물과 AI 대화 흔적을 함께 분석해 더 정교한 진단 결과를 생성하기 위한 선택입니다.

- Demo-safe heuristic evaluator
  공개 저장소와 라이브 심사 환경에서 API 키 없이도 앱이 안정적으로 동작하도록 하기 위해 탑재했습니다. 이는 배포 안정성과 재현성을 높이는 장치입니다.

### 각 AI 도구별 활용 전략

#### 1. Codex 기반 개발 에이전트

- 페르소나
  제품 기획과 구현을 동시에 밀어붙이는 시니어 풀스택 엔지니어

- 핵심 전략
  아이디어 선정 단계에서 먼저 문제 정의를 좁히고, 그 뒤 MVP를 즉시 구현하는 방식으로 진행했습니다. 흔한 AI 튜터/채점기 대신 "AI 시대에 사라진 학습 이해의 증거"를 잡는 방향으로 기획을 고정했습니다.

- 산출물
  제품 화면, 진단 로직, API 라우트, README, AI 리포트 초안

#### 2. 런타임 AI 분석 레이어

- 페르소나
  교강사를 돕는 학습 증거 감사관

- 핵심 전략
  단순히 요약하거나 칭찬하지 않고, 다음 기준으로 진단하도록 설계했습니다.
  - 핵심 개념을 실제로 언급하고 있는가
  - 요구사항이 바뀌면 해결 방식을 전이할 수 있는가
  - 왜 그런 결정을 했는지 설명 가능한가
  - AI에 답을 요구했는지, 힌트와 검증을 요청했는지

- 데이터 흐름
  입력: 과제 설명 + 제출물 + AI 대화 흔적  
  처리: 증거 점수 산출 + 위험 신호 분류 + 방어 질문 생성 + 개입 액션 제안  
  출력: 단일 학습자 진단 카드 + 코호트 우선순위

#### 3. Demo-safe evaluator

- 페르소나
  심사 환경용 안정성 레이어

- 핵심 전략
  라이브 모델 호출이 실패하거나 API 키가 없는 상황에서도 동일한 제품 경험을 제공하도록 설계했습니다.

### 토큰 낭비를 최소화하고 유지보수성 및 재현성을 높이기 위한 전략

- 모델 호출을 단일 API 라우트로 분리해 프롬프트와 스키마를 중앙 관리합니다.
- 모델이 없을 때는 deterministic fallback을 사용해 데모 실패를 방지합니다.
- 결과 포맷을 구조화된 JSON 기반으로 고정해 후속 UI 렌더링을 단순화합니다.
- 공개 저장소에 비밀키가 필요하지 않도록 `.env.example`만 제공하고 실제 키는 배포 환경 변수로 주입합니다.
- 제품 핵심 개념과 제출용 설명은 `docs/proofloop-design.md`, `docs/AI_REPORT_DRAFT.md`에 분리해 유지보수성을 높였습니다.
